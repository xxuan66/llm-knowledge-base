# MCP 架构与核心概念

## 什么是 MCP？

**MCP (Model Context Protocol)** 是一个开源标准协议，用于连接 AI 应用程序与外部系统。

更实用一点说，MCP 的价值在于把“模型如何连接工具和资源”这件事从一堆零散私有接线，逐步变成可复用、可标准化的接口层。

想象一下 MCP 就像 AI 应用的 **USB-C 接口** —— 正如 USB-C 为电子设备提供了标准化的连接方式，MCP 为 AI 应用连接外部系统提供了标准化协议。

### MCP 能实现什么？

- 🤖 AI 助手可以访问你的 Google Calendar 和 Notion，成为更个性化的智能助手
- 💻 Claude Code 可以根据 Figma 设计生成完整的 Web 应用
- 📊 企业聊天机器人可以连接组织的多个数据库，让用户通过对话分析数据
- 🎨 AI 模型可以在 Blender 中创建 3D 设计并通过 3D 打印机输出

### 为什么 MCP 重要？

| 角色 | 收益 |
|------|------|
| **开发者** | 减少开发时间和复杂度，构建或集成 AI 应用更简单 |
| **AI 应用/助手** | 获得丰富的数据源、工具和应用生态系统，增强能力 |
| **最终用户** | 获得更强大的 AI 应用，可以访问你的数据并代表你执行操作 |

最重要的不是背术语，而是理解：

## MCP 架构详解

### 核心参与者

MCP 采用**客户端 - 服务器架构**：

```
┌─────────────────────────────────────────┐
│      MCP Host (AI 应用程序)              │
│  ┌──────────┐  ┌──────────┐            │
│  │  Client 1│  │  Client 2│  ...       │
│  └────┬─────┘  └────┬─────┘            │
└───────┼─────────────┼──────────────────┘
        │             │
        ▼             ▼
   ┌─────────┐   ┌─────────┐
   │ Server A│   │ Server B│
   │ (本地)   │   │ (远程)   │
   └─────────┘   └─────────┘
```

#### 三个关键角色

1. **MCP Host（主机）**
   - AI 应用程序（如 Claude Desktop、VS Code、Cursor）
   - 协调管理一个或多个 MCP 客户端

2. **MCP Client（客户端）**
   - 维护与 MCP 服务器的连接
   - 从服务器获取上下文信息供 Host 使用
   - 每个服务器对应一个专用客户端

3. **MCP Server（服务器）**
   - 提供上下文数据的程序
   - 可以是本地服务（如文件系统）或远程服务（如 Sentry）

### MCP 的两个层次

MCP 由两个层次组成：

#### 1. 数据层（Data Layer）

**核心：** 定义客户端和服务器之间的通信协议

- **协议基础：** JSON-RPC 2.0
- **生命周期管理：** 连接初始化、能力协商、连接终止
- **服务器功能：**
  - Tools（工具）：AI 可执行的操作
  - Resources（资源）：上下文数据源
  - Prompts（提示）：交互模板
- **客户端功能：**
  - Sampling：请求 LLM 完成
  - Elicitation：请求用户输入
  - Logging：调试和监控

#### 2. 传输层（Transport Layer）

**核心：** 管理通信通道和认证

支持两种传输机制：

| 传输方式 | 用途 | 特点 |
|----------|------|------|
| **Stdio** | 本地进程通信 | 使用标准输入/输出，无网络开销，性能最优 |
| **Streamable HTTP** | 远程服务器通信 | 使用 HTTP POST + SSE 流式传输，支持 OAuth 认证 |

## MCP 核心原语（Primitives）

这是 MCP 最重要的概念！定义了客户端和服务器能互相提供什么。

### 服务器提供的三大原语

#### 1. Tools（工具）

**定义：** AI 应用可以调用执行的操作函数

**示例：**
- 文件操作（读取、写入、删除）
- API 调用（天气查询、数据库查询）
- 系统命令执行

**相关方法：**
- `tools/list` - 发现可用工具
- `tools/call` - 执行工具

#### 2. Resources（资源）

**定义：** 为 AI 应用提供上下文信息的数据源

**示例：**
- 文件内容
- 数据库记录
- API 响应数据
- 系统状态信息

**相关方法：**
- `resources/list` - 列出可用资源
- `resources/read` - 读取资源内容

#### 3. Prompts（提示）

**定义：** 帮助结构化与语言模型交互的可重用模板

**示例：**
- 系统提示词模板
- Few-shot 示例
- 特定任务的交互流程

**相关方法：**
- `prompts/list` - 列出可用提示
- `prompts/get` - 获取提示内容

### 客户端提供的原语

#### 1. Sampling（采样）

允许服务器请求客户端的 AI 应用提供 LLM 完成结果。

**用途：** 服务器需要使用 LLM 但希望保持模型无关性

**方法：** `sampling/complete`

#### 2. Elicitation（诱导）

允许服务器请求用户提供额外信息或确认操作。

**用途：** 获取用户输入、确认敏感操作

**方法：** `elicitation/request`

#### 3. Logging（日志）

服务器向客户端发送日志消息用于调试和监控。

## MCP 生命周期管理

MCP 是**有状态协议**，需要生命周期管理来协商能力。

### 初始化流程

```json
// 客户端发送初始化请求
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "initialize",
  "params": {
    "protocolVersion": "2025-06-18",
    "capabilities": {
      "elicitation": {}
    },
    "clientInfo": {
      "name": "example-client",
      "version": "1.0.0"
    }
  }
}

// 服务器响应
{
  "jsonrpc": "2.0",
  "id": 1,
  "result": {
    "protocolVersion": "2025-06-18",
    "capabilities": {
      "tools": {
        "listChanged": true
      },
      "resources": {}
    },
    "serverInfo": {
      "name": "example-server",
      "version": "1.0.0"
    }
  }
}
```

### 初始化交换的关键目的

1. **协议版本协商** - 确保双方使用兼容的协议版本
2. **能力发现** - 声明各自支持的功能（tools、resources、prompts 等）
3. **身份交换** - 提供标识和版本信息用于调试

### 通知机制（Notifications）

MCP 支持实时通知，实现服务器和客户端之间的动态更新。

**示例场景：**
- 服务器的可用工具列表发生变化
- 新资源可用
- 长时间操作需要进度更新

**特点：**
- 使用 JSON-RPC 2.0 通知消息（不需要响应）
- 支持 `tools/list_changed` 等通知类型

## 一个现实判断

MCP 很重要，但它不是业务逻辑的替代品。它更像是连接层。真正系统是否可用，仍然取决于权限设计、工具边界、错误处理、日志与调试以及数据源质量。

## 生态系统支持

MCP 获得广泛的客户端和服务器支持：

### 支持的客户端

| 客户端 | 类型 |
|--------|------|
| **Claude Desktop** | AI 助手 |
| **ChatGPT** | AI 助手 |
| **Visual Studio Code** | 开发工具 |
| **Cursor** | AI 代码编辑器 |
| **MCPJam** | MCP 客户端工具 |

### 官方参考服务器

- **Filesystem Server** - 本地文件系统访问
- **Sentry MCP Server** - 错误监控集成
- **Database Servers** - 各种数据库连接
- **Weather Server** - 天气 API 集成

## 总结

MCP 通过标准化的协议层和灵活的传输层，为 AI 应用与外部系统的连接提供了统一标准。

**关键要点：**

✅ MCP = AI 应用的 USB-C 接口  
✅ 客户端 - 服务器架构，Host 管理多个 Client  
✅ 数据层（JSON-RPC 2.0）+ 传输层（Stdio/HTTP）  
✅ 三大服务器原语：Tools、Resources、Prompts  
✅ 支持实时通知和动态能力发现  
✅ 广泛的生态系统支持
