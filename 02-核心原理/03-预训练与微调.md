# 预训练与微调

## 学习目标

学完本节后，你应该能够：

- 分清预训练、微调、指令微调、对齐分别在做什么
- 知道今天真正常见的大模型适配方式有哪些
- 理解为什么很多业务问题不一定要靠微调解决
- 在项目里判断什么时候该选提示词、RAG、微调或工具调用

## 先建立主线

很多人在学这部分时容易混淆：

- 预训练
- 微调
- 指令微调
- RLHF
- RAG

更好的理解方式是把它们放进同一条主线里：

1. **预训练**：给模型打通用底子
2. **微调 / 指令微调**：让模型更适合某类任务或交互方式
3. **对齐**：让模型更符合人类使用预期
4. **系统增强**：用检索、工具、工作流弥补模型参数本身的边界

## 1. 预训练在解决什么问题

### 1.1 定义与目标

预训练是在大规模数据上让模型学习一般性的语言规律、知识模式和序列建模能力。它更像是在造“底座”，而不是直接造“可上线产品”。

你可以把预训练理解成：

- 模型学会怎么处理语言
- 模型获得广泛但不稳定的知识压缩能力
- 模型具备后续适配各种任务的可能性

### 1.2 预训练任务类型

**1.2.1 因果语言建模（Causal LM）**
- **目标**：预测下一个词（Next Token Prediction）
- **代表模型**：GPT 系列
- **特点**：单向，自回归生成

**1.2.2 掩码语言建模（Masked LM）**
- **目标**：预测被掩码的词
- **代表模型**：BERT 系列
- **特点**：双向上下文理解

**1.2.3 序列到序列建模（Seq2Seq）**
- **目标**：输入序列 → 输出序列
- **代表模型**：T5、BART
- **特点**：编码器-解码器结构

### 1.3 预训练数据

预训练数据的质量会深刻影响模型边界。不是“数据越多越好”这么简单，还包括：

- 清洗是否到位
- 去重是否充分
- 领域分布是否合理
- 代码、数学、多语言等能力是否被覆盖

## 2. 微调在解决什么问题

### 2.1 定义与流程

微调是在已有预训练模型基础上做进一步适配。它的重点不是“从零学会语言”，而是“把已有能力往某个方向拉”。

典型目标包括：

- 让模型更适合某个任务
- 让模型更适合某个领域语料
- 让模型更符合某种交互风格

### 2.2 微调类型

**2.2.1 全参数微调**
- **方法**：更新所有模型参数
- **优势**：性能最大化
- **劣势**：计算成本高，需要大量存储

**2.2.2 参数高效微调（PEFT）**
- **方法**：只更新部分参数
- **代表技术**：
  - **LoRA**：低秩适配器
  - **Adapter**：插入小型适配模块
  - **Prompt Tuning**：优化提示参数
- **优势**：节省计算和存储资源

### 2.3 指令微调

指令微调是把模型从“会续写文本”拉向“能遵循人类指令”的关键步骤。对今天多数聊天式产品体验来说，这一步比“参数再大一点”更直接。

## 3. 对齐不是锦上添花，而是可用性的关键

### 3.1 RLHF 的角色

RLHF 可以理解成一种“让模型更像人类希望它那样工作”的方法。它解决的问题包括：

- 输出更符合偏好
- 更像在对话，而不是机械续写
- 减少明显有害或无关输出

### 3.2 常见三阶段流程
1. **监督微调（SFT）**：在演示数据上微调
2. **奖励模型训练**：学习人类偏好
3. **强化学习优化**：使用 PPO 算法优化

### 3.3 现实中的变化

今天很多系统不一定严格按早期 RLHF 叙事实现，实际还会混合：

- SFT
- 偏好优化
- DPO / 类 DPO 方法
- 安全策略
- 系统提示词和后处理

所以你要理解它的目的，而不是只记一个算法名。

## 4. 预训练、微调、RAG 不要混

很多项目里真正最重要的判断不是“怎么微调”，而是：

> 这个问题到底该不该微调？

一个简单判断方式：

- **知识经常变化**：优先考虑 RAG
- **输出风格或格式固定**：提示词 / 轻量适配先行
- **需要稳定的领域行为模式**：再考虑微调
- **需要调用外部动作**：优先工具调用或 Agent

## 一个实用对比

| 方式 | 更适合解决什么问题 |
|------|-------------------|
| 提示词优化 | 输出风格、结构和任务说明 |
| RAG | 最新知识、私有知识、可追溯知识 |
| 微调 | 稳定行为模式、领域表达习惯、任务适配 |
| 工具调用 | 需要真实执行动作或读取外部系统 |

### 预训练任务实现

```python
import torch
import torch.nn as nn
import torch.nn.functional as F
from transformers import AutoTokenizer, AutoModelForCausalLM

class MaskedLanguageModeling(nn.Module):
    """掩码语言建模（BERT 预训练任务）"""
    
    def __init__(self, vocab_size, d_model=768):
        super().__init__()
        self.embedding = nn.Embedding(vocab_size, d_model)
        self.transformer = nn.TransformerEncoderLayer(d_model, nhead=12)
        self.mlm_head = nn.Linear(d_model, vocab_size)
    
    def forward(self, input_ids, attention_mask=None):
        # 获取嵌入
        embeddings = self.embedding(input_ids)
        
        # Transformer 编码
        if attention_mask is not None:
            # 创建注意力掩码
            attention_mask = attention_mask.bool()
        
        # 这里简化处理，实际需要完整的 Transformer
        output = embeddings  # 简化
        
        # 预测掩码位置
        logits = self.mlm_head(output)
        return logits

class CausalLanguageModeling(nn.Module):
    """因果语言建模（GPT 预训练任务）"""
    
    def __init__(self, vocab_size, d_model=768):
        super().__init__()
        self.embedding = nn.Embedding(vocab_size, d_model)
        self.transformer = nn.TransformerEncoderLayer(d_model, nhead=12)
        self.lm_head = nn.Linear(d_model, vocab_size)
    
    def create_causal_mask(self, seq_len):
        """创建因果掩码"""
        mask = torch.triu(torch.ones(seq_len, seq_len), diagonal=1).bool()
        return ~mask
    
    def forward(self, input_ids):
        # 获取嵌入
        embeddings = self.embedding(input_ids)
        
        # 创建因果掩码
        seq_len = input_ids.size(1)
        causal_mask = self.create_causal_mask(seq_len).to(input_ids.device)
        
        # Transformer 编码（简化）
        output = embeddings  # 简化
        
        # 预测下一个词
        logits = self.lm_head(output)
        return logits

# 测试预训练任务
vocab_size = 10000
model = CausalLanguageModeling(vocab_size)

batch_size, seq_len = 2, 10
input_ids = torch.randint(0, vocab_size, (batch_size, seq_len))
logits = model(input_ids)

print(f"输入形状: {input_ids.shape}")
print(f"输出 logits 形状: {logits.shape}")
```

### LoRA 微调实现

```python
import torch
import torch.nn as nn

class LoRALayer(nn.Module):
    """LoRA 适配器层"""
    
    def __init__(self, original_layer, rank=8, alpha=16):
        super().__init__()
        self.original_layer = original_layer
        self.rank = rank
        self.alpha = alpha
        
        # 冻结原始层参数
        for param in self.original_layer.parameters():
            param.requires_grad = False
        
        # 创建 LoRA 参数
        in_features = original_layer.in_features
        out_features = original_layer.out_features
        
        self.lora_A = nn.Parameter(torch.randn(in_features, rank))
        self.lora_B = nn.Parameter(torch.zeros(rank, out_features))
        self.scaling = alpha / rank
    
    def forward(self, x):
        # 原始层输出
        original_output = self.original_layer(x)
        
        # LoRA 路径
        lora_output = x @ self.lora_A @ self.lora_B * self.scaling
        
        # 合并输出
        return original_output + lora_output

class LoRALinear(nn.Module):
    """带 LoRA 的线性层"""
    
    def __init__(self, in_features, out_features, rank=8, alpha=16):
        super().__init__()
        self.original_linear = nn.Linear(in_features, out_features)
        self.lora = LoRALayer(self.original_linear, rank, alpha)
    
    def forward(self, x):
        return self.lora(x)

# 测试 LoRA
input_dim, output_dim = 512, 256
x = torch.randn(2, 10, input_dim)

# 普通线性层
linear = nn.Linear(input_dim, output_dim)
print(f"普通线性层参数量: {sum(p.numel() for p in linear.parameters()):,}")

# LoRA 线性层
lora_linear = LoRALinear(input_dim, output_dim, rank=8)
lora_params = sum(p.numel() for p in lora_linear.parameters() if p.requires_grad)
total_params = sum(p.numel() for p in lora_linear.parameters())
print(f"LoRA 线性层总参数量: {total_params:,}")
print(f"LoRA 可训练参数量: {lora_params:,} ({lora_params/total_params*100:.1f}%)")

# 前向传播
output = lora_linear(x)
print(f"输出形状: {output.shape}")
```

### 指令微调数据准备

```python
class InstructionDataset:
    """指令微调数据集"""
    
    def __init__(self, tokenizer, max_length=512):
        self.tokenizer = tokenizer
        self.max_length = max_length
        
        # 示例指令数据
        self.data = [
            {
                "instruction": "解释什么是机器学习",
                "input": "",
                "output": "机器学习是人工智能的一个分支，它使计算机能够从数据中学习，而无需进行明确编程。"
            },
            {
                "instruction": "将以下句子翻译成英文",
                "input": "今天天气很好",
                "output": "The weather is very nice today."
            },
            {
                "instruction": "写一首关于春天的诗",
                "input": "",
                "output": "春风拂面柳丝长，\n百花齐放满园香。\n燕子归来寻旧巢，\n人间处处好风光。"
            }
        ]
    
    def format_instruction(self, instruction, input_text="", output=""):
        """格式化指令"""
        if input_text:
            prompt = f"指令：{instruction}\n输入：{input_text}\n输出："
        else:
            prompt = f"指令：{instruction}\n输出："
        return prompt + output
    
    def tokenize(self, example):
        """分词处理"""
        text = self.format_instruction(
            example["instruction"],
            example["input"],
            example["output"]
        )
        
        # 编码
        encoding = self.tokenizer(
            text,
            truncation=True,
            max_length=self.max_length,
            padding="max_length",
            return_tensors="pt"
        )
        
        # 创建标签（输入部分设为 -100，不计算损失）
        labels = encoding["input_ids"].clone()
        # 这里简化处理，实际需要更复杂的逻辑
        
        return {
            "input_ids": encoding["input_ids"].squeeze(),
            "attention_mask": encoding["attention_mask"].squeeze(),
            "labels": labels.squeeze()
        }
    
    def __len__(self):
        return len(self.data)
    
    def __getitem__(self, idx):
        return self.tokenize(self.data[idx])

# 使用示例
from transformers import AutoTokenizer
tokenizer = AutoTokenizer.from_pretrained("bert-base-uncased")
dataset = InstructionDataset(tokenizer)
print(f"数据集大小: {len(dataset)}")
print(f"第一个样本: {dataset[0]}")
```
