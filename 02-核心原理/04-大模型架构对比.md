# 大模型架构对比

## 学习目标

学完本节后，你应该能够：

- 分清三类主流架构各自最适合什么任务
- 理解为什么今天大多数通用 LLM 选择 Decoder-only
- 知道不同架构在理解、生成、转换任务中的取舍
- 建立一套更实用的架构选择思路

## 先说结论

如果只看今天的大语言模型主流路线，Decoder-only 已经成为最常见选择。但这并不意味着其他架构没有价值，而是说明：

- 通用生成任务对 Decoder-only 更友好
- 统一训练和部署路径更简单
- 对话和代码生成场景推动了这一路线的普及

不过，在理解类任务、序列转换任务里，Encoder-only 和 Encoder-Decoder 依旧非常重要。

## 核心知识点

## 1. 三大架构概述

#### 1.1 Encoder-only（编码器-only）
**代表模型**：BERT、RoBERTa、ALBERT

**架构特点**：
- 只使用 Transformer 编码器
- 双向上下文理解
- 输出是整个序列的表示

**典型任务**：
- 文本分类
- 命名实体识别
- 句子对分类
- 语义相似度计算

#### 1.2 Decoder-only（解码器-only）
**代表模型**：GPT 系列、LLaMA、ChatGLM

**架构特点**：
- 只使用 Transformer 解码器
- 单向自回归生成
- 掩码自注意力机制

**典型任务**：
- 文本生成
- 对话系统
- 代码生成
- 创作任务

#### 1.3 Encoder-Decoder（编码器-解码器）
**代表模型**：T5、BART、mT5

**架构特点**：
- 编码器理解输入
- 解码器生成输出
- 交叉注意力连接两者

**典型任务**：
- 机器翻译
- 文本摘要
- 问答系统
- 序列转换

## 2. 详细对比分析

#### 2.1 训练目标对比

| 架构 | 训练目标 | 优势 | 劣势 |
|------|---------|------|------|
| Encoder-only | 掩码语言建模 | 双向理解，适合理解任务 | 不擅长生成 |
| Decoder-only | 因果语言建模 | 生成能力强，训练简单 | 单向理解 |
| Encoder-Decoder | 序列到序列建模 | 适合转换任务 | 参数量较大 |

#### 2.2 计算效率对比

| 架构 | 预训练速度 | 推理速度 | 内存占用 |
|------|-----------|---------|---------|
| Encoder-only | 快（并行） | 中等 | 中等 |
| Decoder-only | 快（并行） | 慢（自回归） | 中等 |
| Encoder-Decoder | 较慢 | 中等 | 较高 |

#### 2.3 应用场景对比

**Encoder-only 最适合**：
- 文本分类和情感分析
- 语义搜索和检索
- 知识图谱构建
- 特征提取和表示学习

**Decoder-only 最适合**：
- 开放域对话
- 创意写作和内容生成
- 代码补全和生成
- 零样本/少样本学习

**Encoder-Decoder 最适合**：
- 机器翻译
- 文本摘要和改写
- 数据到文本生成
- 问答系统

## 3. 架构演进趋势

#### 3.1 混合架构
- **UniLM**：统一三种架构
- **GLM**：自回归空白填充
- **PaLM 2**：改进的编码器-解码器

#### 3.2 效率优化
- **稀疏注意力**：处理长序列
- **量化技术**：降低计算需求
- **知识蒸馏**：模型压缩

#### 3.3 多模态扩展
- **视觉-语言模型**：CLIP、DALL-E
- **音频-语言模型**：Whisper、AudioGPT
- **多模态大模型**：GPT-4V、Gemini

## 4. 一个更实用的选择指南

#### 4.1 根据任务类型选择

**理解类任务** → Encoder-only
- 需要深度理解文本语义
- 输出是分类标签或表示
- 不需要生成新文本

**生成类任务** → Decoder-only
- 需要生成连贯文本
- 开放域或创意性任务
- 需要保持生成一致性

**转换类任务** → Encoder-Decoder
- 输入和输出都是序列
- 需要精确的映射关系
- 如翻译、摘要等

#### 4.2 根据资源限制选择

**计算资源有限**：
- 选择小参数量模型
- 考虑量化或蒸馏版本
- 使用参数高效微调

**延迟要求高**：
- 优先考虑 Encoder-only
- 使用缓存和优化推理
- 考虑模型蒸馏

## 一个现实判断

真实项目里，模型架构选择很少是纯理论问题，通常还会受这些因素影响：

- 生态是否成熟
- 部署框架是否支持
- 社区资料是否丰富
- 微调和量化工具链是否齐全

所以最终决策往往是“任务适配 + 工程现实”的共同结果。

## 代码示例

### 架构性能对比实验

```python
import torch
import torch.nn as nn
from transformers import AutoModel, AutoModelForCausalLM, AutoModelForSeq2SeqLM
import time

class ArchitectureBenchmark:
    """架构性能对比"""
    
    def __init__(self):
        self.models = {}
        self.results = {}
    
    def load_models(self):
        """加载不同架构的模型"""
        # Encoder-only (BERT)
        self.models['encoder'] = AutoModel.from_pretrained("bert-base-uncased")
        
        # Decoder-only (GPT-2)
        self.models['decoder'] = AutoModelForCausalLM.from_pretrained("gpt2")
        
        # Encoder-Decoder (T5)
        self.models['enc-dec'] = AutoModelForSeq2SeqLM.from_pretrained("t5-small")
        
        print(f"已加载 {len(self.models)} 个模型")
    
    def benchmark_inference(self, input_text, max_length=50):
        """推理性能测试"""
        from transformers import AutoTokenizer
        
        for name, model in self.models.items():
            print(f"\n测试 {name} 架构:")
            
            # 加载对应的 tokenizer
            if name == 'encoder':
                tokenizer = AutoTokenizer.from_pretrained("bert-base-uncased")
                inputs = tokenizer(input_text, return_tensors="pt")
                
                start_time = time.time()
                with torch.no_grad():
                    outputs = model(**inputs)
                inference_time = time.time() - start_time
                
                print(f"  输出形状: {outputs.last_hidden_state.shape}")
                
            elif name == 'decoder':
                tokenizer = AutoTokenizer.from_pretrained("gpt2")
                inputs = tokenizer(input_text, return_tensors="pt")
                
                start_time = time.time()
                with torch.no_grad():
                    outputs = model.generate(
                        **inputs,
                        max_length=max_length,
                        do_sample=False
                    )
                inference_time = time.time() - start_time
                
                generated = tokenizer.decode(outputs[0], skip_special_tokens=True)
                print(f"  生成长度: {len(generated.split())} 词")
                
            elif name == 'enc-dec':
                tokenizer = AutoTokenizer.from_pretrained("t5-small")
                inputs = tokenizer(
                    "translate English to French: " + input_text,
                    return_tensors="pt"
                )
                
                start_time = time.time()
                with torch.no_grad():
                    outputs = model.generate(
                        **inputs,
                        max_length=max_length
                    )
                inference_time = time.time() - start_time
                
                translated = tokenizer.decode(outputs[0], skip_special_tokens=True)
                print(f"  翻译结果: {translated}")
            
            self.results[name] = {
                'time': inference_time,
                'model_size': sum(p.numel() for p in model.parameters())
            }
            print(f"  推理时间: {inference_time:.3f}s")
    
    def compare_results(self):
        """对比结果"""
        print("\n" + "="*60)
        print("架构性能对比结果")
        print("="*60)
        
        for name, result in self.results.items():
            print(f"\n{name}:")
            print(f"  参数量: {result['model_size']:,}")
            print(f"  推理时间: {result['time']:.3f}s")
        
        # 找出最快的
        fastest = min(self.results.items(), key=lambda x: x[1]['time'])
        print(f"\n最快架构: {fastest[0]} ({fastest[1]['time']:.3f}s)")

# 使用示例
benchmark = ArchitectureBenchmark()
benchmark.load_models()
benchmark.benchmark_inference("The quick brown fox jumps over the lazy dog")
benchmark.compare_results()
```

### 架构选择决策树

```python
def recommend_architecture(task_type, requirements):
    """根据任务类型推荐架构"""
    
    recommendations = {
        '理解类任务': {
            '架构': 'Encoder-only',
            '模型': ['BERT', 'RoBERTa', 'ALBERT'],
            '适用任务': ['文本分类', '实体识别', '情感分析'],
            '优点': '双向理解，准确率高',
            '缺点': '不擅长生成'
        },
        '生成类任务': {
            '架构': 'Decoder-only',
            '模型': ['GPT', 'LLaMA', 'ChatGLM'],
            '适用任务': ['对话', '创作', '代码生成'],
            '优点': '生成能力强，连贯性好',
            '缺点': '单向理解，计算量大'
        },
        '转换类任务': {
            '架构': 'Encoder-Decoder',
            '模型': ['T5', 'BART', 'mT5'],
            '适用任务': ['翻译', '摘要', '问答'],
            '优点': '适合序列到序列转换',
            '缺点': '参数量较大'
        }
    }
    
    if task_type not in recommendations:
        print(f"未知任务类型: {task_type}")
        return None
    
    rec = recommendations[task_type]
    print(f"\n推荐架构: {rec['架构']}")
    print(f"推荐模型: {', '.join(rec['模型'])}")
    print(f"适用任务: {', '.join(rec['适用任务'])}")
    print(f"优点: {rec['优点']}")
    print(f"缺点: {rec['缺点']}")
    
    # 根据具体需求调整
    if requirements.get('low_latency', False):
        print("\n⚠️  注意：低延迟需求，建议使用较小的模型或量化版本")
    if requirements.get('long_context', False):
        print("\n⚠️  注意：长上下文需求，考虑使用支持长序列的模型")
    
    return rec

# 使用示例
task_type = '生成类任务'
requirements = {
    'low_latency': True,
    'long_context': False
}
recommend_architecture(task_type, requirements)
```
