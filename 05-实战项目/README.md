# 实战项目

> 这一部分的重点不是“堆 demo”，而是把前面学过的概念落到完整应用里。

## 这一章怎么读

如果你已经完成基础入门、核心原理和进阶应用，来到这一章时，应该重点关注：

- 一个项目为什么这样设计
- 它的关键风险点是什么
- 哪些问题属于模型问题，哪些属于系统问题
- 如果要上线，哪里需要补强

## 阅读建议

### 如果你是第一次做 LLM 项目

推荐顺序：

1. 对话机器人
2. 知识库问答系统
3. 微调专属模型
4. 多 Agent 协作系统

### 如果你在做企业应用

优先看：

- 知识库问答系统
- 模型评测
- 微调专属模型

### 如果你在做复杂流程自动化

优先看：

- 多 Agent 协作系统
- Agent 智能体
- MCP 协议

## 一个总原则

不要把项目章节当成“复制即上线”的方案，更适合把它们当成：

- 架构参考
- 能力边界示例
- 常见取舍说明
- 进入真实系统前的练习场
