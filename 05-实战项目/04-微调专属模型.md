# 实战项目四：微调专属模型

> 使用 LoRA/QLoRA 技术微调一个属于你的专属大语言模型，让模型学会你的领域知识和表达风格。

这个项目最容易踩的坑，是一上来就默认“要做微调”。实际上，一个成熟项目在进入微调前，通常已经先完成了：

- 任务边界定义
- 数据整理
- 提示词验证
- 是否需要 RAG 的判断
- 基线模型选择

## 项目目标

完成一次完整的 LLM 微调流程：

1. **数据准备** — 收集和清洗训练数据
2. **模型选择** — 选择合适的基础模型
3. **微调训练** — 使用 LoRA/QLoRA 进行参数高效微调
4. **模型评估** — 评测微调效果
5. **模型部署** — 将微调后的模型上线使用

## 什么时候真的值得做微调？

| 场景 | 说明 |
|------|------|
| 领域行为模式 | 让模型更稳定地按某类任务风格输出 |
| 表达风格 | 让模型更接近特定语气、文风、模板要求 |
| 格式遵循 | 让模型更稳定地遵循固定结构 |
| 任务优化 | 针对特定类型任务做行为适配 |

如果只是知识更新问题，通常应优先考虑 RAG，而不是微调。

## 技术路线

### 选择一：LoRA（推荐）

**Low-Rank Adaptation** — 参数高效的微调方法：

- **原理：** 冻结原模型，只训练低秩矩阵
- **优点：** 显存占用小（7B 模型只需 ~6GB 显存），训练快
- **工具：** PEFT + Transformers

### 选择二：QLoRA

**Quantized LoRA** — 量化 + LoRA：

- **原理：** 将基础模型量化到 4-bit，再叠加 LoRA
- **优点：** 显存占用极小（7B 模型只需 ~4GB 显存）
- **工具：** bitsandbytes + PEFT

### 选择三：全量微调

- **优点：** 效果最好，充分利用模型能力
- **缺点：** 显存需求大（7B 模型需 ~28GB）
- **适用：** 有充足 GPU 资源时

### 2. 数据收集

```python
def prepare_data(documents: list, output_file: str):
    """将文档转化为训练数据"""
    from openai import OpenAI
    client = OpenAI()
    
    dataset = []
    
    for doc in documents:
        # 使用 GPT-4 生成问答对
        prompt = f"""根据以下内容，生成 5 个高质量的问答对：

内容：
{doc[:2000]}

格式：
Q: 问题
A: 答案
"""
        
        response = client.chat.completions.create(
            model="gpt-4",
            messages=[{"role": "user", "content": prompt}]
        )
        
        qa_pairs = parse_qa_pairs(response.choices[0].message.content)
        dataset.extend(qa_pairs)
    
    # 保存为 JSONL
    with open(output_file, "w", encoding="utf-8") as f:
        for item in dataset:
            f.write(json.dumps(item, ensure_ascii=False) + "\n")
    
    return dataset
```

### 3. 数据质量检查

数据质量通常比训练技巧更重要。很多失败的微调项目，问题根源都不在学习率，而在：

- 样本风格不一致
- 输出质量不稳定
- 样本目标定义模糊
- 数据量太少但目标太大

```python
def validate_dataset(file_path: str):
    """检查数据集质量"""
    issues = []
    
    with open(file_path, "r") as f:
        for i, line in enumerate(f):
            item = json.loads(line)
            
            # 检查格式
            if "messages" not in item:
                issues.append(f"Line {i}: Missing 'messages' field")
            
            # 检查角色
            roles = [m["role"] for m in item["messages"]]
            if "system" not in roles:
                issues.append(f"Line {i}: Missing system message")
            
            # 检查长度
            total_len = sum(len(m["content"]) for m in item["messages"])
            if total_len > 4000:
                issues.append(f"Line {i}: Content too long ({total_len} chars)")
    
    print(f"Found {len(issues)} issues")
    return issues
```

## 训练流程

进入训练前，建议先准备一个简单基线：

1. 不微调，直接用基座模型测试
2. 用提示词测试
3. 必要时加少量检索
4. 如果这些都仍然不稳定，再进入微调

### QLoRA 训练代码

```python
from transformers import (
    AutoModelForCausalLM,
    AutoTokenizer,
    TrainingArguments,
    BitsAndBytesConfig
)
from peft import LoraConfig, get_peft_model, prepare_model_for_kbit_training
from trl import SFTTrainer
from datasets import load_dataset

# 1. 配置 4-bit 量化
bnb_config = BitsAndBytesConfig(
    load_in_4bit=True,
    bnb_4bit_quant_type="nf4",
    bnb_4bit_compute_dtype="bfloat16",
    bnb_4bit_use_double_quant=True
)

# 2. 加载基础模型
model_id = "meta-llama/Llama-3-8b"  # 或其他开源模型
model = AutoModelForCausalLM.from_pretrained(
    model_id,
    quantization_config=bnb_config,
    device_map="auto",
    trust_remote_code=True
)
tokenizer = AutoTokenizer.from_pretrained(model_id)
tokenizer.pad_token = tokenizer.eos_token

# 3. 准备 LoRA 配置
lora_config = LoraConfig(
    r=16,                    # LoRA rank
    lora_alpha=32,           # LoRA scaling
    target_modules=["q_proj", "v_proj", "k_proj", "o_proj"],  # 目标模块
    lora_dropout=0.05,
    bias="none",
    task_type="CAUSAL_LM"
)

# 4. 应用 LoRA
model = prepare_model_for_kbit_training(model)
model = get_peft_model(model, lora_config)

# 5. 加载数据集
dataset = load_dataset("json", data_files="train.jsonl", split="train")

# 6. 配置训练参数
training_args = TrainingArguments(
    output_dir="./results",
    num_train_epochs=3,
    per_device_train_batch_size=4,
    gradient_accumulation_steps=4,
    learning_rate=2e-4,
    fp16=True,
    logging_steps=10,
    save_steps=100,
    warmup_steps=100,
    optim="paged_adamw_8bit"
)

# 7. 创建训练器
trainer = SFTTrainer(
    model=model,
    train_dataset=dataset,
    dataset_text_field="text",
    max_seq_length=2048,
    tokenizer=tokenizer,
    args=training_args,
    peft_config=lora_config
)

# 8. 开始训练
trainer.train()

# 9. 保存模型
trainer.model.save_pretrained("./my-finetuned-model")
tokenizer.save_pretrained("./my-finetuned-model")
```

### 训练参数调优

| 参数 | 推荐值 | 说明 |
|------|-------|------|
| `r` (LoRA rank) | 8-64 | 越大能力越强，但显存消耗增加 |
| `lora_alpha` | r × 2 | 通常设为 rank 的 2 倍 |
| `learning_rate` | 1e-4 ~ 5e-4 | LoRA 可以用较高的学习率 |
| `epochs` | 3-5 | 过多会过拟合 |
| `batch_size` | 4-16 | 根据显存调整 |

## 模型评估

### 1. 基准测试

```python
def evaluate_model(model, tokenizer, test_set):
    """评估模型性能"""
    results = []
    
    for item in test_set:
        # 构建提示词
        prompt = format_prompt(item["input"])
        
        # 生成回复
        inputs = tokenizer(prompt, return_tensors="pt").to(model.device)
        outputs = model.generate(**inputs, max_new_tokens=512)
        response = tokenizer.decode(outputs[0], skip_special_tokens=True)
        
        # 与参考答案对比
        score = compute_similarity(response, item["expected"])
        results.append(score)
    
    avg_score = sum(results) / len(results)
    print(f"平均得分: {avg_score:.2f}")
    return avg_score
```

## 常见问题

### Q: 训练时显存不足怎么办？
- 减小 `per_device_train_batch_size`
- 增加 `gradient_accumulation_steps`
- 使用 QLoRA 而非 LoRA
- 减小 `max_seq_length`

### Q: 训练后模型输出质量下降？
- 检查数据质量
- 降低学习率
- 增加训练数据
- 检查是否有过拟合（用验证集评估）

### Q: 如何判断需要多少数据？
- 简单任务：50-100 条
- 中等任务：500-1000 条
- 复杂任务：5000+ 条
