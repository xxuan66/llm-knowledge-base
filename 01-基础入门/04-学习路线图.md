# LLM 学习路线图

## 学习目标

学完本节后，你应该能够：

- 按自己的背景选择更合适的学习顺序
- 分清“先学什么”和“后学什么”
- 避免一上来就陷入热点名词和模型榜单
- 为自己安排一个可执行的 4-12 周学习计划

## 先说结论

学习 LLM 最容易踩的坑，不是资料太少，而是顺序错了。

典型错误包括：

- 一开始就追最新模型榜单
- 还没理解 token 和上下文窗口，就去做 Agent
- 还没搞清推理与评测，就急着堆框架
- 只会调 API，但不知道什么时候该用 RAG、什么时候该微调

所以更合理的路线不是“从最火的开始”，而是：

1. 先建立框架
2. 再理解原理
3. 再进入工程和应用
4. 最后再做模型选型与系统设计

## 推荐学习阶段

### 第一阶段：建立整体认知

建议先读：

- `01-什么是LLM.md`
- `02-发展历程.md`
- `03-核心概念.md`

这一阶段的目标不是会写代码，而是回答下面几个问题：

- LLM 到底是什么
- 它为什么和传统 NLP 不一样
- token、上下文窗口、预训练、微调、推理分别是什么意思
- 今天的大模型系统为什么经常要和检索、工具、工作流结合

### 第二阶段：补齐核心原理

建议重点读：

- `02-核心原理/01-Transformer架构.md`
- `02-核心原理/02-注意力机制.md`
- `02-核心原理/03-预训练与微调.md`

这一阶段的目标是：

- 知道 Transformer 为什么成为主流架构
- 理解自注意力为什么是关键突破
- 理解“预训练 - 对齐 - 推理”这条主线

### 第三阶段：进入工程实践

建议重点读：

- `03-动手实践/01-从零搭建LLM.md`
- `03-动手实践/02-Tokenizer实现.md`
- `03-动手实践/04-微调与量化.md`

这一阶段的目标是：

- 知道模型训练和推理到底涉及哪些组件
- 理解为什么工程上总在谈 LoRA、量化、显存、吞吐、延迟
- 从“会调用模型”进一步走向“知道系统为什么这样搭”

### 第四阶段：学习真实应用

建议重点读：

- `04-进阶应用/01-RAG检索增强.md`
- `04-进阶应用/02-Agent智能体.md`
- `04-进阶应用/03-模型评测.md`

这一阶段的目标是：

- 分清哪些问题更适合 RAG，哪些更适合微调
- 理解 Agent 为什么不是简单的多轮对话
- 知道评测为什么是上线前必须做的事情

### 第五阶段：回到模型与系统选型

建议重点读：

- `06-主流开源模型/`
- `07-MCP 协议/`

这一阶段的目标是：

- 不只会看模型名字，而是能理解模型家族差异
- 理解模型如何与工具、外部系统、协议连接

## 按背景给建议

### 路线 A：零基础或跨行读者

建议节奏：6-10 周

顺序：

1. 基础入门
2. 核心原理
3. 进阶应用
4. 动手实践
5. 主流模型

重点：

- 不要一开始追求自己训练模型
- 先把概念和系统视角建立起来

### 路线 B：有 Python / 后端开发基础

建议节奏：4-8 周

顺序：

1. 基础入门中的关键章节
2. 核心原理
3. 动手实践
4. RAG / Agent / 评测
5. 模型选型

重点：

- 尽快把理解迁移到工程判断上
- 学会看成本、效果、复杂度的取舍

### 路线 C：有机器学习背景

建议节奏：3-6 周

顺序：

1. 快速浏览基础入门
2. 深读核心原理
3. 补工程实践
4. 聚焦应用与系统设计

重点：

- 不要只停留在模型训练视角
- 补齐部署、推理、RAG、评测、工作流连接这些系统问题

## 一个更实用的时间安排

### 每周 5-7 小时

适合长期稳定学习：

- 2 小时读原理
- 2 小时读应用或案例
- 1-3 小时做笔记和小实验

### 每周 8-12 小时

适合集中进阶：

- 3 小时补理论
- 3 小时读代码或文档
- 2-6 小时做实践和复盘

## 学习时最重要的产出

不要只追求“看完”，更建议每个阶段产出这些东西：

1. 一份自己的术语表
2. 一份模型能力边界判断清单
3. 一份 RAG / 微调 / 工具调用的选择标准
4. 至少一个可运行的小项目或实验记录

## 常见问题

### 没有 GPU 能不能学

可以。

前期大部分学习重点其实在概念、推理、应用结构和系统理解上，不需要一开始就投入 GPU。真正需要算力的部分，可以后置到：

- 小模型体验
- 推理实验
- 轻量微调

### 数学一般，会不会学不动

可以学。

先把概念和直觉建立起来，再补公式细节。对于多数应用型学习者来说，先理解“为什么这样设计”比先推完整公式更重要。

### 资料太多怎么选

先用这套仓库建立主框架，再去看外部资料。不要一开始就把自己埋进“课程、论文、博客、视频、开源仓库”里。

### 学习进度追踪器

```python
class LearningTracker:
    """学习进度追踪器"""
    
    def __init__(self):
        self.progress = {
            '基础入门': {'01-什么是LLM': False, '02-发展历程': False, 
                        '03-核心概念': False, '04-学习路线图': False},
            '核心原理': {'01-Transformer架构': False, '02-注意力机制': False,
                        '03-预训练与微调': False, '04-大模型架构对比': False},
            '动手实践': {'01-从零搭建LLM': False, '02-Tokenizer实现': False,
                        '03-预训练流程': False, '04-微调与量化': False},
            '进阶应用': {'01-RAG检索增强': False, '02-Agent智能体': False,
                        '03-模型评测': False, '04-前沿技术': False},
            '实战项目': {'01-对话机器人': False, '02-知识库问答': False,
                        '03-多Agent协作': False, '04-微调专属模型': False}
        }
    
    def mark_complete(self, stage, chapter):
        """标记完成"""
        if stage in self.progress and chapter in self.progress[stage]:
            self.progress[stage][chapter] = True
            print(f"✅ {stage}/{chapter} 已完成")
        else:
            print(f"❌ 未找到: {stage}/{chapter}")
    
    def get_progress(self):
        """获取进度统计"""
        total = 0
        completed = 0
        for stage in self.progress.values():
            for status in stage.values():
                total += 1
                if status:
                    completed += 1
        return completed, total
    
    def print_report(self):
        """打印进度报告"""
        completed, total = self.get_progress()
        percentage = (completed / total) * 100
        
        print(f"\n{'='*50}")
        print(f"学习进度报告")
        print(f"{'='*50}")
        print(f"总进度: {completed}/{total} ({percentage:.1f}%)")
        print(f"{'='*50}")
        
        for stage, chapters in self.progress.items():
            stage_completed = sum(chapters.values())
            stage_total = len(chapters)
            stage_pct = (stage_completed / stage_total) * 100
            print(f"\n{stage}: {stage_completed}/{stage_total} ({stage_pct:.0f}%)")
            for chapter, status in chapters.items():
                icon = "✅" if status else "⬜"
                print(f"  {icon} {chapter}")

# 使用示例
tracker = LearningTracker()
tracker.mark_complete('基础入门', '01-什么是LLM')
tracker.mark_complete('基础入门', '02-发展历程')
tracker.mark_complete('核心原理', '01-Transformer架构')
tracker.print_report()
```

### 学习计划生成器

```python
def generate_study_plan(background, weeks_available):
    """根据背景和时间生成学习计划"""
    
    plans = {
        'zero': {
            '基础入门': 2,
            '核心原理': 4,
            '动手实践': 8,
            '进阶应用': 4,
            '实战项目': 8
        },
        'python': {
            '基础入门': 1,
            '核心原理': 3,
            '动手实践': 6,
            '进阶应用': 4,
            '实战项目': 8
        },
        'ml': {
            '基础入门': 1,
            '核心原理': 2,
            '动手实践': 3,
            '进阶应用': 2,
            '实战项目': 8
        }
    }
    
    plan = plans.get(background, plans['zero'])
    
    # 根据可用时间调整
    total_weeks = sum(plan.values())
    if weeks_available < total_weeks:
        scale = weeks_available / total_weeks
        for stage in plan:
            plan[stage] = max(1, int(plan[stage] * scale))
    
    print(f"\n学习计划 (背景: {background}, 可用时间: {weeks_available}周)")
    print("="*50)
    current_week = 1
    for stage, weeks in plan.items():
        end_week = current_week + weeks - 1
        print(f"{stage}: 第{current_week}-{end_week}周 ({weeks}周)")
        current_week = end_week + 1
    
    return plan

# 示例：Python基础，有20周时间
generate_study_plan('python', 20)
```
