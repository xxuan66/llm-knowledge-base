# LLM 发展历程

## 学习目标

学完本节后，你应该能够：

- 说清大语言模型是如何一步步演化到今天的
- 理解每个阶段真正重要的突破是什么
- 分清“参数变大”只是表象，背后的关键变化有哪些
- 用发展的视角理解今天为什么会出现开源模型、RAG、Agent、长上下文和工具调用

## 先给一个整体判断

LLM 的发展并不是一条单纯“参数越来越大”的直线，而是几条主线叠加的结果：

1. 表示学习从离散特征走向稠密向量
2. 序列建模从 RNN 走向 Transformer
3. 训练范式从任务定制走向大规模预训练
4. 模型使用方式从“单任务模型”走向“通用能力底座”
5. 工程重点从“能不能训出来”转向“如何高效推理、如何接系统、如何控制质量”

理解这几条主线，比死记模型发布时间更重要。

## 第一阶段：统计语言模型时代

在早期，自然语言处理更多依赖人工特征和统计方法。典型思路是：

- 根据前面出现过的词，估计下一个词的概率
- 使用 n-gram、最大熵等方法做局部概率建模

这一阶段的优点是概念简单、实现直观，但问题也很明显：

- 很难捕捉长距离依赖
- 数据稀疏问题严重
- 泛化能力有限

这阶段的核心贡献，不是模型本身有多强，而是把“语言可以建模”为后续发展打下了基础。

## 第二阶段：神经网络表示学习出现

随后，词向量和神经网络方法逐渐成为主流。

代表性变化包括：

- Word2Vec：让词具有分布式向量表示
- GloVe：结合全局统计信息改进表示能力
- RNN / LSTM：开始尝试更自然地处理序列

这一阶段的重要意义在于：

- 语言不再只是离散符号，而被表示成连续向量
- 模型开始能学习更丰富的上下文关系

但 RNN/LSTM 仍然有明显瓶颈：

- 并行性差
- 长序列训练困难
- 对超长依赖支持有限

## 第三阶段：Transformer 成为分水岭

2017 年，`Attention Is All You Need` 提出 Transformer，几乎重写了后续大模型路线。

真正关键的不只是“换了个新架构”，而是：

- 用自注意力替代循环结构
- 提高并行训练能力
- 更容易扩展到更大规模数据和参数

随后出现的几个关键节点：

- 2018：BERT 让“预训练 + 微调”成为主流范式
- 2019：GPT-2 展示生成式模型的强潜力

从这个阶段开始，语言模型不再只是 NLP 里的一类任务模型，而开始向“通用语言能力”演化。

## 第四阶段：大模型能力爆发

2020-2022 年是能力明显跃迁的阶段。

代表节点包括：

- GPT-3：展示 few-shot / in-context learning 的强大效果
- InstructGPT：指令微调让模型更“可用”
- ChatGPT：对话交互与对齐体验大幅提升

这一阶段最重要的变化不是只有规模，而是：

- 模型开始具备更强的通用交互能力
- “对齐”成为和“预训练”同等重要的议题
- 用户与模型的接口从任务输入升级为提示词 / 对话

换句话说，模型开始从研究对象变成产品底层能力。

## 第五阶段：开源、工程化与系统化

2023 年之后，生态重点进一步变化。

一边是开源模型迅速发展，例如：

- Llama 系列
- Qwen 系列
- Mistral 系列
- DeepSeek 系列
- GLM 系列

另一边是工程重点越来越明确：

- 量化和蒸馏降低推理成本
- LoRA / QLoRA 降低微调门槛
- vLLM、SGLang 等推理框架优化吞吐与延迟
- RAG、Agent、工作流、MCP 等把模型接入真实系统

这意味着今天的大模型竞争，已经不只是“谁参数更大”，而是：

- 谁更容易部署
- 谁更适合某个场景
- 谁更容易接系统
- 谁的成本、速度、效果更平衡

## 一个更有用的时间线

```
统计建模
   ↓
神经网络表示学习
   ↓
Transformer 成为主干架构
   ↓
预训练模型成为主流范式
   ↓
大模型能力爆发
   ↓
指令微调与对齐成为产品关键
   ↓
开源模型快速发展
   ↓
推理优化、RAG、Agent、MCP 进入系统层
```

## 关键阶段对比

| 阶段 | 代表变化 | 真正重要的意义 |
|------|----------|------------------|
| 统计模型 | n-gram、最大熵 | 语言可以被概率建模 |
| 神经表示学习 | Word2Vec、GloVe、LSTM | 语言表示进入向量时代 |
| Transformer | 自注意力、并行训练 | 大规模扩展成为可能 |
| 预训练范式 | BERT、GPT-2 | 通用模型开始替代任务专用模型 |
| 能力爆发 | GPT-3、ChatGPT | 提示词和对话成为主交互方式 |
| 开源与工程 | Llama、Qwen、DeepSeek | 模型真正进入可部署、可选型、可系统集成阶段 |

### 模拟 n-gram 语言模型

```python
from collections import defaultdict, Counter
import random

class NGramModel:
    """简化版 n-gram 语言模型"""
    
    def __init__(self, n=2):
        self.n = n
        self.ngrams = defaultdict(Counter)
    
    def train(self, text):
        """训练模型"""
        words = text.lower().split()
        for i in range(len(words) - self.n + 1):
            context = tuple(words[i:i+self.n-1])
            next_word = words[i+self.n-1]
            self.ngrams[context][next_word] += 1
    
    def generate(self, start_words, length=20):
        """生成文本"""
        result = list(start_words)
        context = tuple(start_words[-(self.n-1):])
        
        for _ in range(length):
            if context in self.ngrams:
                # 根据概率选择下一个词
                candidates = self.ngrams[context]
                total = sum(candidates.values())
                r = random.randint(1, total)
                cumulative = 0
                for word, count in candidates.items():
                    cumulative += count
                    if cumulative >= r:
                        next_word = word
                        break
            else:
                next_word = random.choice(list(set().union(*[c.keys() for c in self.ngrams.values()])))
            
            result.append(next_word)
            context = tuple(result[-(self.n-1):])
        
        return ' '.join(result)

# 使用示例
text = "the quick brown fox jumps over the lazy dog the quick brown fox is very quick"
model = NGramModel(n=2)
model.train(text)

print("生成示例：", model.generate(["the", "quick"], length=10))
```

## 学习时要抓住的三件事

1. **不要只记模型名字**
重要的是理解每次跃迁背后的技术方向。

2. **不要只看参数规模**
很多关键变化来自训练范式、对齐方法、推理工程和系统接入。

3. **把“模型发展”放进“系统演化”里看**
今天的大模型已经不只是模型论文问题，而是完整系统问题。
