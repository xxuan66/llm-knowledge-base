# 主流开源模型

> 本章节不是模型新闻榜单，而是帮助你建立“如何选模型”的判断框架。

今天看开源模型，最容易犯的错是只盯着排行榜、参数量和单点 benchmark。真正有用的选型视角通常包括：

- 任务类型
- 语言与领域适配
- 上下文长度
- 部署成本
- 推理速度
- 工具调用与系统接入能力
- 社区生态与维护活跃度

## 本章适合怎么读

### 如果你只想先建立选型概念

先读本页，再根据自己的重点跳到对应模型家族。

### 如果你已经准备部署

优先看：

- 任务选择
- 硬件选择
- 部署方式对比

### 如果你是做中文应用

优先比较：

- Qwen
- GLM
- DeepSeek
- 其他国产模型

### 如果你更看重本地部署与工程稳定性

优先比较：

- Llama
- Mistral
- Qwen

## 快速选择指南

### 一个更实用的选型顺序

建议按这个顺序判断：

1. 先看任务类型
2. 再看语言与领域偏好
3. 再看部署资源和成本上限
4. 再看长上下文、代码、工具调用等专项能力
5. 最后再用 benchmark 做候选之间的细比

### 不要只按参数量选择

参数量是重要信息，但不是唯一信息。很多真实项目里，更关键的是：

- 响应速度
- 推理成本
- 长上下文是否稳定
- 中文、代码、工具调用能力是否符合需求

下面这张表只适合作为粗略起点。

| 规模 | 推荐模型 | 显存需求 (FP16) | 适用场景 |
|------|---------|---------------|---------|
| 小模型 (<10B) | Llama3-8B, Qwen2.5-7B, Mistral 7B | 14-16 GB | 本地部署、边缘设备 |
| 中模型 (10-50B) | Qwen2.5-32B, Yi-34B, Mixtral 8x7B | 60-80 GB | 企业应用、专业任务 |
| 大模型 (>50B) | Llama3-70B, Qwen2.5-72B, DeepSeek-V2 | 140-280 GB | 云端服务、高性能需求 |

### 按任务选择

| 任务 | 推荐模型 | 理由 |
|------|---------|------|
| 通用对话 | Llama3-8B, Qwen2.5-7B | 平衡性能与资源 |
| 中文任务 | Qwen2.5, GLM-4, Baichuan2 | 中文优化 |
| 代码生成 | DeepSeek-Coder, Qwen2.5-Coder | 代码能力强化 |
| 长文档 | Qwen2.5-128K, Yi-200K, GLM-4 | 长上下文支持 |
| 多模态 | Qwen-VL, GLM-4V | 图像理解 |
| 工具调用 | Qwen2.5, ChatGLM3, InternLM2 | 原生 Function Calling |

### 按硬件选择

| 硬件配置 | 推荐模型 | 量化建议 |
|---------|---------|---------|
| RTX 3060 12G | Llama3-8B, Qwen2.5-7B, Mistral 7B | INT4 可运行 13B |
| RTX 3090 24G | Llama3-70B(INT4), Qwen2.5-32B | INT4 推荐 |
| RTX 4090 24G | 同上，推理更快 | INT4/FP16 |
| 2×A100 80G | Llama3-70B, Qwen2.5-72B | FP16/INT8 |
| 4×A100 80G | DeepSeek-V2, Mixtral 8x22B | FP16 |

## 看 benchmark 时要注意什么

基准分数很有用，但不能直接替代真实业务判断。原因包括：

- benchmark 与业务分布不一致
- 同一个模型在不同推理框架上的表现差异很大
- 长上下文、工具调用、结构化输出常常无法通过单一榜单体现

所以看 benchmark 的正确姿势是：

1. 先看是否在大方向上符合预期
2. 再看是否适合你的任务类型
3. 最后结合自己的部署条件做测试

## 部署方式对比

### Ollama（推荐新手）

```bash
# 安装
curl -fsSL https://ollama.ai/install.sh | sh

# 运行模型
ollama run llama3:8b
ollama run qwen2.5:7b
ollama run mistral:7b
ollama run deepseek-coder:6.7b
```

**优点：** 一键安装，自动下载，简单易用  
**缺点：** 自定义选项较少

### vLLM（推荐生产）

```bash
pip install vllm

python -m vllm.entrypoints.api_server \
    --model meta-llama/Meta-Llama-3-8B-Instruct \
    --port 8000 \
    --max-model-len 8192
```

**优点：** 高吞吐，PagedAttention，生产级  
**缺点：** 配置较复杂

### llama.cpp（推荐本地）

```bash
# 编译
git clone https://github.com/ggerganov/llama.cpp
cd llama.cpp && make

# 量化
python convert-hf-to-gguf.py model
./quantize model-f16.gguf model-q4_k_m.gguf Q4_K_M

# 运行
./main -m model-q4_k_m.gguf -p "Hello" -n 256
```

**优点：** CPU 可运行，量化效果好，跨平台  
**缺点：** 需要转换模型格式

## 一个更实用的选型顺序

建议按这个顺序做决策：

1. 先确定任务类型
2. 再确定语言和领域偏好
3. 再看硬件和预算
4. 再看是否需要长上下文 / 工具调用 / 多模态
5. 最后才在候选模型里比 benchmark

## 模型选择决策树

```
开始
│
├─ 需要本地运行？
│  ├─ 是 → 显存多少？
│  │       ├─ <8GB → Yi-6B, Qwen2.5-3B (INT4)
│  │       ├─ 8-16GB → Llama3-8B, Qwen2.5-7B, Mistral 7B
│  │       └─ 16-24GB → Llama3-70B(INT4), Qwen2.5-32B(INT4)
│  │
│  └─ 否 → 继续
│
├─ 主要用途？
│  ├─ 中文任务 → Qwen2.5, GLM-4, Baichuan2
│  ├─ 代码生成 → DeepSeek-Coder, Qwen2.5-Coder
│  ├─ 长文档 → Yi-34B-200K, Qwen2.5-128K
│  └─ 通用 → Llama3, Mixtral
│
├─ 性能要求？
│  ├─ 最高 → Llama3.1-405B, DeepSeek-V3, Qwen2.5-72B
│  ├─ 平衡 → Llama3-70B, Qwen2.5-32B, Mixtral 8x22B
│  └─ 轻量 → Llama3-8B, Qwen2.5-7B, Mistral 7B
│
└─ 预算限制？
   ├─ 免费开源 → 全部上述模型
   ├─ 可接受 API → GPT-4, Claude, Gemini
   └─ 企业采购 → 联系模型厂商获取商用授权
```

## 学习建议

### 入门路线

1. **第一阶段（1-2 周）**
   - 了解 LLM 基础概念
   - 使用 Ollama 运行 Llama3-8B
   - 尝试简单对话和问答

2. **第二阶段（2-4 周）**
   - 学习 Prompt Engineering
   - 尝试不同模型对比
   - 了解量化和部署

3. **第三阶段（1-2 月）**
   - 学习 RAG 技术
   - 尝试模型微调（LoRA）
   - 构建实际应用

4. **第四阶段（持续）**
   - 关注新模型发布
   - 参与开源项目
   - 探索前沿技术

## 使用这章时要避免的两个误区

1. 看到新模型就立刻替换当前方案
2. 把“最强模型”误当成“最适合你的模型”

真正成熟的做法是：

- 先选一个稳定可用的基线模型
- 再针对关键任务做对比测试
- 最后根据成本、速度、效果决定是否迁移
